#  AI 开发规则文档

## 一、总则
- 所有 AI 辅助开发活动必须严格遵守本规则，确保与项目实际运行需求保持一致，维护项目的整体架构完整性。
- 每次会话请确保：全面检查分析项目，确保完美修复或实现用户提出问题，如果任务需要执行功能优化，请务必与类似先进项目进行对比分析，并 以先进项目为标准实现需求，并确保所有调整不会存在或导致任何错误及冗余。
- 会话必须使用全程中文，每次次会话必须执行：发现语法错误、冗余代码、重复定义、垃圾代码、无效代码等内容需要及时清理。
- 项目测试使用浏览器工具测试，不要通过脚本测试，避免测试过程中出现异常情况。
## 二、核心开发规则

### 1. 禁止创建测试文件与模拟数据
- 严禁创建任何测试文件、演示数据、模拟代码及相关功能
- 如因特殊需求必须创建测试内容，测试完成后必须立即清理所有测试文件和代码，确保不影响项目正式环境
- 禁止在正式代码中包含任何仅用于测试目的的注释、标记或代码块、测试用例等。

### 2. 架构与功能集成规范
- 项目apps目录内包含每个应用模块目录内严格禁止出现templates目录或者其他前端模板文件。发现即清理或迁移到根目录的templates目录内。
- 项目功能模块必须在apps目录内，不可以在其他位置出现。发现即清理或迁移到apps目录内。
- 新增功能时，必须深入了解项目整体架构，确保新功能与现有系统深度融合
- 严格遵循 Django 项目的应用结构设计，新功能应合理划分到相应的应用模块中
- 杜绝出现功能孤岛、数据孤岛及重复数据模型等问题
- 对于跨应用的数据交互，必须通过已定义的接口或公共服务进行，禁止直接访问其他应用的数据库模型
- 新增数据模型时，必须先检查是否存在可复用的现有模型，避免数据结构重复

### 3. 前端设计一致性要求
- 前端设计必须严格遵循项目统一的设计规范和风格指南
- 禁止出现配色方案、视觉风格孤立等影响项目整体视觉一致性的设计元素
- 所有页面必须使用项目已定义的 CSS 样式、组件库和图标系统
- 新创建的页面或组件在视觉效果上必须与现有页面保持高度一致
- 如需使用新的前端技术或框架，必须事先评估其与项目现有技术栈的兼容性
- 项目所有二级页面或下级页面必须使用从右侧弹出并占据右侧工作区80%方式展现，不能使用其他方式展现。
### 4. 持续错误检测与修正
- 保持会话持续活跃，在代码编写、运行等全流程中进行实时错误检测
- 一旦发现代码语法错误、逻辑错误或运行时异常，必须立即进行修正
- 确保所有修改的代码能够正常编译和运行，不会引入新的错误
- 对于复杂功能的实现，应分步骤验证，确保每一步都符合预期结果
- 修正错误时，必须理解问题的根本原因，避免仅修复表面症状

## 三、代码规范
- 严格遵守 Python/Django 代码风格指南，保持代码可读性和可维护性
- 所有代码必须添加适当的注释，特别是复杂逻辑和关键算法部分
- 变量、函数、类的命名必须清晰明了，遵循项目已有的命名规范
- 避免编写过长的函数或方法，复杂逻辑应进行合理拆分
- 确保代码的健壮性，添加必要的异常处理和边界条件检查

## 四、安全规范
- 严格遵守项目的安全规范，保护用户数据和系统安全
- 禁止在代码中硬编码敏感信息，如密码、API 密钥等
- 遵循最小权限原则，限制代码对系统资源的访问权限
- 对用户输入进行严格验证和过滤，防止安全漏洞
