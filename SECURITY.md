# 安全策略

感谢您关注 DT 企业智能管理系统的安全性！

## 报告安全漏洞

如果您发现任何安全漏洞，请通过以下方式报告：

1. **不要**在 GitHub Issues 中公开报告安全漏洞
2. 发送邮件至：support@dtcall.cn
3. 我们会在 24 小时内确认收到您的报告
4. 我们会尽快修复并发布安全更新

## 安全最佳实践

### 生产环境部署

- 始终使用 HTTPS 加密传输
- 定期更新系统和依赖包
- 使用强密码策略
- 启用审计日志
- 定期备份数据库

### 敏感信息保护

- 不要将配置文件（包含密钥、密码等）提交到版本库
- 使用环境变量存储敏感配置
- 定期轮换 API 密钥和数据库密码

### 访问控制

- 遵循最小权限原则
- 定期审查用户权限
- 启用双因素认证（推荐）

## 依赖安全

我们使用 GitHub Dependabot 监控项目依赖的安全更新，请确保及时更新依赖包。
