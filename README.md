# DT企业智能管理系统

## 项目概述

DT企业智能管理系统是一套基于Django 5.2框架开发的企业级数字化管理平台，旨在为企业提供全方位的智能化管理解决方案。该系统集成了用户管理、客户关系管理、合同管理、财务管理、生产管理、项目管理、办公自动化、人工智能、任务管理、消息管理、企业管理、公共模块、部门管理、岗位管理、企业网盘、个人办公、系统管理、蜘蛛爬虫、工作流、新财务模块、审批模块、首页等二十二个核心功能模块，通过深度融合人工智能技术，帮助企业实现业务流程的智能化升级，提升管理效率，降低运营成本。

系统采用模块化架构设计，各功能模块之间高内聚低耦合，既可独立运行又能协同工作，便于企业根据实际需求进行灵活配置和扩展。系统同时支持私有化部署和云端部署，能够适应不同规模和类型企业的管理需求。无论是中小型企业的日常管理，还是大型集团的复杂业务流程，DT企业智能管理系统都能提供成熟稳定的解决方案。

## 技术架构

### 后端技术栈

DT企业智能管理系统的后端采用业界成熟的Django框架构建，充分利用其强大的ORM系统、完善的认证授权机制以及丰富的第三方扩展生态。系统要求Python 3.8及以上版本运行环境，推荐使用Python 3.10或3.11以获得最佳性能体验。Django 5.2版本带来了诸多新特性和性能优化，包括更高效的数据库查询、改进的异步支持以及增强的安全特性，为系统的稳定运行提供了坚实基础。

在数据库层面，系统使用PostgreSQL 10及以上版本作为主数据库，PostgreSQL以其卓越的数据完整性保障、丰富的数据类型支持以及强大的扩展能力而著称。系统完全支持PostgreSQL 10及更高版本的所有特性，并针对大规模数据场景进行了专项优化。对于缓存和会话存储，系统集成Redis 6及以上版本，通过高性能的内存缓存机制显著提升系统响应速度，有效降低数据库访问压力。

系统提供完整的RESTful API接口，基于Django REST Framework构建，支持多种认证方式，包括SimpleJWT令牌认证、Session认证以及OAuth2.0认证。API文档采用OpenAPI规范生成，便于第三方系统集成和前端开发对接。所有API接口均经过严格的权限校验，确保数据安全。

### 前端技术栈

前端界面采用LayUI 2.8+作为主要UI框架，LayUI是一款经典的开源前端组件库，以其简洁优雅的设计风格和丰富的组件生态受到国内开发者的广泛认可。系统充分利用LayUI提供的表单、表格、弹层、导航等组件，构建了功能完善、交互流畅的用户界面。同时，系统集成了ECharts图表库，用于数据可视化展示，包括各类统计报表、趋势分析图表以及实时监控大屏。

在文件处理方面，系统内置了完善的Office文档在线预览功能，支持Word、Excel、PowerPoint、PDF等常见文档格式的浏览器内预览，无需下载即可在线查阅。此外，系统还支持图片、音视频等多种媒体格式的在线预览，满足企业日常办公的多样化需求。

### 第三方服务集成

DT企业智能管理系统深度集成了多种主流云存储服务，包括阿里云OSS、腾讯云COS、百度云BOS、七牛云Kodo、AWS S3以及WebDAV协议存储。企业可以根据现有IT基础设施和成本考量灵活选择云存储服务商，系统提供统一的文件上传、下载、预览接口，屏蔽底层存储差异，简化开发和使用复杂度。

在人工智能领域，系统支持接入多种主流大语言模型和AI服务，包括OpenAI GPT系列、Azure OpenAI、Anthropic Claude、Google Gemini、百度文心一言、阿里通义千问、腾讯混元、DeepSeek以及字节豆包等。同时，系统还支持本地模型部署，满足数据安全要求较高的企业场景。通过统一的AI接口封装，开发者可以便捷地在业务场景中集成AI能力。

语音服务方面，系统集成了语音转文字（STT）功能，支持多种语音识别服务，可应用于会议记录、语音输入、智能客服等场景。在工业数据采集领域，系统支持MQTT、Modbus、OPC UA等主流工业协议，能够对接各类生产设备和传感器，实现生产数据的实时采集和监控。

## 核心功能模块

### 用户与权限管理模块

用户与权限管理是企业级应用的基础模块，DT系统在这方面提供了完整成熟的解决方案。用户认证支持用户名密码登录、验证码验证以及多因素认证等多种方式，有效保障账户安全。系统采用基于RBAC（角色权限控制）的权限模型，支持细粒度的权限分配，可精确到菜单、按钮乃至数据字段级别。

组织架构管理功能涵盖部门管理、岗位管理和员工档案管理三个维度。部门管理支持树形结构的无限级嵌套，适应复杂的企业组织架构。岗位管理提供了完整的职位体系定义，包括职级、职称、岗位职责等信息的维护。员工档案管理则记录了员工的入职信息、工作经历、教育背景、调动记录、奖惩记录等完整的人事数据。

员工关怀功能是系统的特色模块之一，支持员工生日自动提醒、节日问候推送、定期回访计划制定等功能，体现企业以人为本的管理理念。系统还内置了完整的操作日志记录功能，追踪记录所有用户的关键操作，为安全审计提供数据支撑。

### 客户关系管理模块

客户关系管理是DT系统的核心业务模块之一，提供了从线索获取到成交转化再到售后服务的全生命周期管理能力。客户信息管理支持维护客户的基本资料、联系信息、企业背景等完整档案，并支持自定义字段扩展，满足不同行业客户的差异化需求。

客户公海功能实现了公共客户资源池的管理，支持客户的领取、分配、回收等操作。通过公海机制，系统确保了客户资源在销售团队中的合理流动，避免客户资源的闲置和浪费。客户分类管理提供了多维度的分类体系，包括客户等级、行业分类、来源渠道等，便于销售人员精准定位目标客户。

联系人管理功能维护了客户方的关键联系人信息，包括联系人姓名、职位、联系方式、决策权限等，为商务沟通和关系维护提供支持。跟进记录功能完整记录了每一次客户拜访和沟通的详情，支持文字记录、附件上传以及下次跟进计划制定。

订单管理和合同管理功能实现了销售业务的闭环管理。订单支持从创建到履约的全流程跟踪，包括订单状态流转、物流信息更新、收货确认等环节。合同管理支持合同的起草、审批、签署、归档全流程，并与订单、发票等业务模块实现数据联动。

发票管理功能支持发票的开具、作废以及回款跟踪，与财务模块深度集成。拨号记录功能记录了每一次电话沟通的详情，支持通话状态追踪和拨打结果统计。自定义字段功能允许企业根据自身业务需求，在客户、跟进记录、订单等实体上添加自定义字段，灵活扩展系统能力。

此外，系统还集成了天眼查等数据源的爬虫任务，支持自动获取潜在客户的企业信息，辅助销售团队快速了解客户背景，提高商机转化效率。

### 合同管理模块

合同管理模块提供了销售合同和采购合同的统一管理能力。销售合同支持从起草到归档的完整生命周期管理，包括合同文本编辑、参与人员指派、审批流程流转、电子签署集成以及归档存储等环节。采购合同管理则支持供应商管理、采购条款约定、付款计划制定等功能。

产品管理功能实现了合同与产品服务的关联，支持产品目录维护、单价管理、库存联动等能力。付款管理功能提供了付款计划的制定和回款记录的追踪，支持多期付款、分期收款等复杂场景。供应商管理功能维护了完整的供应商信息库，包括供应商资质、合作历史、评价记录等内容。

### 财务管理模块

财务管理模块涵盖了企业日常财务管理的核心需求，包括报销管理、发票管理、回款管理和财务统计四个主要功能。报销管理支持费用申请、审批、打款的完整流程，涵盖差旅费、业务招待费、办公用品采购等多种费用类型。审批流程可根据企业规模和管理需求灵活配置，支持多级审批、条件分支等复杂场景。

发票管理功能支持进项发票和销项发票的统一管理，实现发票的登记、查验、认证、作废等操作。系统支持与国家税务系统对接，实现发票信息的自动获取和验真。回款管理功能记录了每一笔回款的详细信息，支持账期管理、催收提醒、坏账处理等能力。

财务统计功能提供了多维度的财务报表和分析，包括费用统计、收入统计、利润分析、现金流分析等。系统支持自定义报表配置，满足不同层级管理者的数据需求。所有统计数据均支持图形化展示，便于决策层快速把握财务状况。

### 生产管理模块

生产管理模块面向制造型企业，提供从生产计划到成品入库的完整生产流程管理能力。工序管理功能定义了基本的生产工序，支持工序集的组合配置以及工艺路线的设计，为生产排程提供基础数据。设备管理功能维护了完整的设备档案，包括设备基本信息、技术参数、运行状态、维护保养记录等。

生产计划功能支持生产任务的下达和排程计划的制定，支持插单、改单等灵活调度需求。BOM管理功能实现了物料清单的维护和成本核算，支持多级BOM结构，满足复杂产品的生产需求。数据采集功能通过MQTT、Modbus等工业协议对接生产设备和传感器，实现生产数据的实时采集和统计分析。

质量管理功能支持质量标准的设定和检验记录的维护，包括来料检验、过程检验、成品检验等环节。SOP管理功能维护了标准作业程序文档，并支持执行跟踪和合规检查。领料管理功能实现了领料申请、材料出库、退料处理的完整流程。完工申报功能支持产成品的完工报告和红冲处理。成品入库功能则完成了产品入库的登记和库存更新。

### 项目管理模块

项目管理模块为企业的项目型业务提供支持，实现了从项目立项到结项的完整生命周期管理。项目立项功能支持项目信息的登记、团队成员的指派、里程碑节点的设定等。进度跟踪功能实时展示项目的执行状态，支持与计划进度进行对比分析。

任务管理功能实现了任务的分解、指派和跟踪，支持任务优先级设定、进度更新、工时填报等操作。项目文档管理功能提供了项目相关文档的集中存储和共享，支持版本控制和权限管理。工时记录功能记录了团队成员在项目上的实际投入，支持工时统计和成本核算。

### 办公自动化模块

办公自动化模块提供了企业日常行政办公所需的各项功能。审批流程功能支持自定义审批流程的配置，包括审批类型、审批节点、审批条件、审批人规则等要素，支持会签、或签等多种审批模式。会议管理功能实现了会议室的预订、会议安排的制定以及会议纪要的记录，支持会议通知推送和参会提醒。

消息通知功能支持系统消息、公告通知、邮件通知等多种通知方式，支持按部门、角色、个人等维度进行精准推送。工作日程功能提供了个人日程的安排和提醒设置，支持日程共享和团队协调。审批记录功能保存了所有审批的完整历史，支持追溯查询和统计分析。

### 任务管理与消息管理模块

任务管理模块提供了轻量级的任务管理能力，支持任务的创建、指派、跟踪和完成，适用于跨部门协作和日常工作任务的管理。消息管理模块则实现了系统消息的统一管理和推送，支持消息分类、消息模板、消息阅读状态追踪等功能。

### 企业管理与公共模块

企业管理模块实现了企业主体的信息管理，包括企业基本信息、证照信息、联系方式等的维护。公共模块提供了系统级的公共服务，包括基础数据管理、缓存服务、数据整合等功能，是其他业务模块的公共能力支撑。

### 部门管理与岗位管理模块

部门管理模块实现了企业部门架构的维护，支持部门的增删改查、部门负责人指派、部门间层级关系设置等功能。岗位管理模块提供了岗位体系的管理，包括岗位的创建、岗位描述的编写、任职条件的设定以及岗位与员工关系的维护。

### 企业网盘模块

企业网盘模块为企业提供了安全高效的文件存储和共享服务。文件管理功能支持文件的上传、下载、文件夹的创建和删除、文件的重命名和移动等操作。权限控制功能实现了基于角色和部门的文件访问权限管理，支持只读、读写、管理等多种权限级别。

文件预览功能支持文档、图片、音视频等多种格式的在线预览，无需下载即可查看内容。文件分享功能支持生成分享链接，可设置链接的有效期和访问密码，实现文件的内外部分享。

### 个人办公模块

个人办公模块为每位用户提供个人相关功能的入口，包括个人信息的维护、偏好的设置、我的文件、我的消息等功能入口，帮助用户在系统中管理个人相关的信息和资源。

### 系统管理模块

系统管理模块提供了系统级的配置和管理能力。系统配置功能支持各种系统参数的设置和功能开关的控制。菜单管理功能实现了系统菜单的配置和权限关联，支持菜单的动态加载和权限过滤。存储配置功能提供了云存储服务的配置管理，支持阿里云OSS、腾讯云COS等多种存储服务商的接入。

服务配置功能用于配置AI服务等外部服务的接入信息，包括API密钥、接口地址、超时时间等参数。系统还提供了完善的数据备份和恢复功能，支持定时自动备份和手动备份，确保数据安全。

### 人工智能模块

人工智能模块是DT系统的核心特色之一，提供了丰富的AI能力支撑。AI模型配置功能支持多种AI服务商的配置管理，包括OpenAI、Azure、Anthropic、Google以及国内主流AI服务商，用户可根据需求灵活选择和切换。

AI工作流功能提供了可视化的工作流设计器，支持拖拽式构建复杂的AI业务流程。工作流支持多种节点类型，包括AI模型节点、数据输入输出节点、条件判断节点、循环节点、延迟节点、Webhook节点等，满足各种业务场景需求。工作流执行监控功能记录了每次执行的详细日志和状态，支持执行历史查询和异常追踪。

AI知识库功能实现了企业知识的管理和智能问答。文档管理支持PDF、Word、Excel、PPT、TXT等格式文档的上传和解析，向量存储基于语义检索技术实现知识的向量化存储和相似度搜索，支持RAG（检索增强生成）模式的知识问答。

AI聊天功能提供了智能对话能力，用户可与AI助手进行自然语言交互，获取信息、完成任务、生成内容等。AI销售策略功能集成了SPIN、FABE、解决方案销售等多种销售方法论，辅助销售人员进行客户沟通和方案呈现。

AI意图识别功能支持询价、比价、投诉、复购等多种客户意图的自动识别，帮助企业快速响应客户需求。AI情绪分析功能分析客户反馈的情绪倾向，为服务质量评估和客户关系管理提供数据支持。AI合规检查功能实现敏感词检测和内容合规审核，降低企业合规风险。AI自动触发功能支持基于条件的自动化行动触发，实现业务流程的智能化编排。A/B测试功能支持AI模型效果的对比测试，帮助企业选择最优的AI服务配置。

### 蜘蛛爬虫模块

蜘蛛爬虫模块提供了数据采集任务的管理能力，支持配置爬虫任务、定时执行、数据存储等功能。系统已集成天眼查等数据源的爬虫模板，可自动获取企业工商信息、股东信息、经营异常等数据，为客户开发和风险评估提供数据支撑。

### 工作流模块

工作流模块提供了业务流程的工作流引擎支持，支持业务流程的建模、执行、监控和优化。引擎遵循WFMC标准，提供流程定义、流程实例管理、任务分配、流程监控等核心功能，可与审批流程、项目流程等业务场景深度集成。

### 新财务模块

新财务模块在原有财务模块基础上进行了功能增强和体验优化，提供了更现代化的财务管理界面和更丰富的财务分析能力，与主财务模块形成互补，满足企业多样化的财务管理需求。

## 快速开始

### 环境要求

在部署DT企业智能管理系统之前，请确保您的环境满足以下要求。操作系统方面，系统支持Windows Server 2016及以上版本、CentOS 7及以上版本、Ubuntu 18.04及以上版本，推荐使用Linux服务器以获得更好的性能和稳定性。Python版本要求3.8或更高，推荐使用3.10或3.11以获得最佳兼容性。数据库要求PostgreSQL 10或更高版本，缓存要求Redis 6或更高版本。

硬件配置方面，推荐服务器至少配备4核CPU、8GB内存、100GB SSD存储空间。对于大型企业部署，建议使用更高配置以确保系统流畅运行。数据库服务器和缓存服务器建议独立部署，以获得更好的性能表现。

### 安装步骤

首先克隆项目代码到本地或服务器。如果使用Git进行版本控制，执行以下命令克隆仓库：

```bash
git clone <repository_url>
cd dtcall
```

如果没有Git环境，也可以直接下载项目压缩包并解压。接下来创建Python虚拟环境，这是Python项目的最佳实践，可以避免依赖冲突：

```bash
python -m venv venv
```

在Windows系统上，激活虚拟环境的命令为：

```bash
venv\Scripts\activate
```

在Linux或macOS系统上，激活命令为：

```bash
source venv/bin/activate
```

激活虚拟环境后，使用pip安装项目依赖：

```bash
pip install -r requirements.txt
```

安装过程可能需要几分钟时间，请耐心等待。如果遇到安装失败的情况，请检查网络连接并确保pip版本是最新的。

接下来配置数据库环境。确保PostgreSQL数据库已创建，并创建了专用的数据库用户和密码。可以通过以下方式配置数据库连接：使用环境变量设置或直接编辑settings.py文件。推荐使用环境变量方式配置敏感信息。

数据库环境变量配置如下：

```bash
export DATABASE_HOST='your_database_host'
export DATABASE_PORT='5432'
export DATABASE_NAME='dtcall'
export DATABASE_USER='dtcall_user'
export DATABASE_PASSWORD='your_secure_password'
```

Redis缓存环境变量配置如下：

```bash
export REDIS_HOST='your_redis_host'
export REDIS_PORT='6379'
export REDIS_PASSWORD='your_redis_password'
```

系统还支持通过环境变量配置Django密钥和其他安全参数：

```bash
export DJANGO_SECRET_KEY='your-very-secret-key-change-this-in-production'
export DEBUG=False
```

如果选择直接编辑配置文件，请打开dtcall/settings.py文件，找到DATABASES和CACHES配置部分，按照实际环境修改连接参数。

完成配置后，运行数据库迁移命令创建所需的表结构：

```bash
python manage.py migrate
```

如果需要初始化默认数据，可以执行数据初始化命令：

```bash
python manage.py init_data
```

迁移完成后，启动Django开发服务器：

```bash
python manage.py runserver 0.0.0.0:8000
```

启动成功后，在浏览器中访问http://your_server_ip:8000即可看到系统登录页面。首次访问时需要使用管理员账户登录，默认管理员账户信息请参考系统初始化文档。

### 生产部署建议

对于生产环境部署，不建议使用Django自带的开发服务器。建议使用Gunicorn或uWSGI作为WSGI服务器，它们是高性能的Python WSGI服务器，能够更好地处理并发请求。

使用Gunicorn部署的典型配置如下：

```bash
pip install gunicorn
gunicorn dtcall.wsgi:application --bind 0.0.0.0:8000 --workers 4 --timeout 120
```

建议配置Nginx作为反向代理服务器和静态文件服务器，Nginx可以提供静态文件的快速响应、请求负载均衡、SSL终端等功能。以下是Nginx配置的关键配置项示例：

```bash
server {
    listen 80;
    server_name your_domain;
    
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /static/ {
        alias /path/to/dtcall/static/;
    }
    
    location /media/ {
        alias /path/to/dtcall/media/;
    }
}
```

生产环境必须启用HTTPS以确保数据传输安全，建议使用Let's Encrypt获取免费的SSL证书。还需要配置定期数据库备份策略，建议每日全量备份并保留最近30天的备份文件。此外，建议部署监控系统监控服务器资源使用情况和应用运行状态，及时发现和处理异常情况。

## 功能特点

DT企业智能管理系统在设计上遵循高内聚低耦合的原则，具备以下显著特点：

**模块化架构**：系统采用模块化设计理念，各功能模块相对独立又有机协作，既便于单独开发和测试，也便于根据企业需求灵活组合配置。模块间通过清晰的接口进行交互，避免了直接耦合带来的维护困难。

**完整API接口**：系统提供完善的RESTful API接口，覆盖所有业务功能，支持第三方系统集成和二次开发。API采用JSON格式进行数据交换，支持分页、排序、筛选等通用功能，并提供完整的接口文档。

**安全可靠**：系统建立了完善的安全体系，包括用户认证、权限控制、数据加密、操作审计等多个层面。权限控制精确到按钮级别，确保用户只能访问授权范围内的数据和功能。数据传输支持HTTPS加密，敏感数据存储支持加密处理。

**高性能**：系统通过多种技术手段确保高性能表现，包括数据库连接池、查询优化、缓存机制、异步任务等。Redis缓存用于存储会话和热点数据，显著减少数据库访问压力。数据库查询经过优化，充分利用索引提升查询效率。

**AI增强**：系统深度集成人工智能能力，提供AI工作流、知识库、智能对话、意图识别、情绪分析等AI功能。通过AI赋能，企业可以大幅提升业务流程的自动化水平和智能化程度，降低人工成本，提高工作效率。

**工业数据采集**：系统支持MQTT、Modbus、OPC UA等主流工业协议，能够对接各种生产设备、传感器和SCADA系统，实现生产数据的实时采集、监控和分析，满足制造型企业的工业互联网需求。

**云存储集成**：系统集成了主流云存储服务，提供统一的文件管理接口，支持阿里云OSS、腾讯云COS、百度云BOS、七牛云Kodo、AWS S3以及WebDAV协议存储，满足企业多样化的存储需求。

## 常见问题

**如何修改系统语言？**

DT系统目前主要面向中文用户，所有界面和文档均为中文。如需支持其他语言，需要进行国际化配置，修改Django的LANGUAGE_CODE设置并提供对应的翻译文件。

**如何进行数据备份？**

系统支持两种备份方式：手动备份通过系统管理界面的备份功能进行；自动备份通过配置定时任务执行备份命令。建议每日进行数据库备份，并将备份文件保存到独立存储位置。

**如何扩展新的功能模块？**

新增功能模块需要遵循Django应用规范，在apps目录下创建新的应用目录，定义模型、视图、URL等组件，然后在settings.py的INSTALLED_APPS中注册新应用。建议参考现有模块的实现方式保持代码风格一致。

**如何对接第三方系统？**

系统提供REST API接口供第三方系统对接，API采用JWT认证方式。详细API文档请参考项目docs目录下的API文档或通过Swagger UI在线查看（开发环境中访问/api/docs/路径）。
